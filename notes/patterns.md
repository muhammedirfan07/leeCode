# Problem Solving Patterns

## 1. Two Pointers
- **When**: sorted array, find pairs, reverse in-place
- **Examples**: Two Sum (sorted), Reverse String, Container With Most Water

## 2. Sliding Window
- **When**: subarray/substring of variable size
- **Examples**: Longest Substring Without Repeating, Max Sum Subarray

## 3. Hash Map / Set
- **When**: need O(1) lookup, counting frequencies
- **Examples**: Two Sum, Anagram check, Contains Duplicate

## 4. Stack
- **When**: matching pairs, next greater element
- **Examples**: Valid Parentheses, Min Stack

## 5. Binary Search
- **When**: sorted array, search space halves each step
- **Examples**: Search in Rotated Array, Find Peak Element

## 6. BFS (Queue)
- **When**: shortest path, level-order traversal
- **Examples**: Word Ladder, Binary Tree Level Order

## 7. DFS (Recursion)
- **When**: explore all paths, backtracking
- **Examples**: Number of Islands, Path Sum, Permutations

## 8. Dynamic Programming
- **When**: overlapping subproblems, optimal substructure
- **Examples**: Climb Stairs, Coin Change

---
## How to approach any problem
1. Read twice. Understand input and output.
2. Ask: what data structure helps here?
3. Think brute force first, then optimize.
4. Write time + space complexity.
5. Test with examples, then edge cases (empty, single, duplicates).
